### 2. 热度评分算法优化

#### 评分公式
```
综合热度 = 基础优先级 + 热度加分
```

#### 热度加分规则

| 热词等级 | 关键词 | 加分 |
|---------|--------|------|
| 超级热词 | 发布、推出、重磅、launch、release、announces、unveils | +6分 |
| 高级热词 | 突破、创新、革命性、breakthrough、innovation | +4分 |
| 中级热词 | 最新、更新、升级、v2、v3、update、upgrade | +3分 |
| 公司热词 | OpenAI、Google、Microsoft、Meta、Anthropic、DeepSeek、NVIDIA | +3分 |
| 技术热词 | GPT-5、GPT-4、Claude 3、LLM、大模型、AGI、Transformer | +2分 |
| 特殊加分 | GitHub热门项目（链接含github.com且标题含stars） | +2分 |

#### 排序机制
1. 计算每条资讯的综合热度分数
2. 按热度从高到低排序
3. 推送数量：15条（原10条）

```python
def calculate_hot_score(news):
    """计算热度分数，基于多维度评分"""
    score = 0
    title = news['title'].lower()
    
    # 检查超级热词
    for keyword in HOT_KEYWORDS['super']:
        if keyword in title:
            score += 6
            break
    
    # 检查高级热词
    for keyword in HOT_KEYWORDS['high']:
        if keyword in title:
            score += 4
            break
    
    # ... 其他评分逻辑
    return round(score, 1)
```

### 8. 常见问题

**Q1: 更新后明天会自动生效吗？**
A: 是的，只要代码推送到 GitHub，明天 8:00 AM 的 GitHub Actions 会自动使用新配置运行。

**Q2: 如何回滚到之前的配置？**
A: 在 GitHub 仓库的 "main.py" 文件历史记录中，点击 "History" 可以查看并恢复到之前的版本。

**Q3: 如果某些资讯源采集失败怎么办？**
A: 系统会自动跳过失败的源，不会影响其他源的采集。可以在 GitHub Actions 日志中查看具体的失败原因。

**Q4: 如何调整推送数量？**
A: 在 main.py 中修改 `MAX_NEWS_TO_PUSH` 参数（当前为 15）。

**Q5: 热度评分可以调整吗？**
A: 可以，修改 `HOT_KEYWORDS` 字典中的关键词和加分值，或调整 `calculate_hot_score` 函数中的评分逻辑。
