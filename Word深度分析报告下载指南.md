## 💡 **常见问题解答**

### **Q1: 为什么报告不能直接推送到飞书？**

**A**: 飞书 Webhook 仅支持发送消息，不支持直接上传文件。要实现文件上传，需要：
1. 配置飞书应用（需要 App ID 和 App Secret）
2. 获取飞书 API 调用权限
3. 使用飞书 API 上传文件

当前方案使用 GitHub Actions Artifacts 存储报告，飞书消息中提供下载指引。

---

### **Q2: 每天都需要手动下载报告吗？**

**A**: 不需要！有3种自动化方式：

1. **GitHub Release**（推荐）：配置后，飞书消息中会包含直接下载链接，点击即可下载
2. **飞书文件上传**（最优）：配置后，报告直接在飞书群中，无需任何操作
3. **定期手动下载**（当前）：每天查看飞书消息，点击链接下载

---

### **Q3: 报告会自动保存历史版本吗？**

**A**: GitHub Actions 默认只保留最近 30 天的运行记录和 Artifacts。如果需要长期保存：

**方案1**：定期手动下载报告到本地
**方案2**：配置 GitHub Release，永久保存所有历史报告
**方案3**：上传到云存储（如阿里云 OSS），按日期归档

---

### **Q4: 如何修改报告的生成时间？**

**A**: 修改 `.github/workflows/daily-8am.yml` 中的 cron 表达式：

```yaml
on:
  schedule:
    - cron: '0 0 * * *'  # UTC 时间 0:00，对应北京时间 8:00
```

**cron 表达式格式**：`分 时 日 月 周`

例如：
- `'0 1 * * *'`：北京时间 9:00
- `'0 2 * * *'`：北京时间 10:00
- `'0 16 * * *'`：北京时间 0:00（次日）
